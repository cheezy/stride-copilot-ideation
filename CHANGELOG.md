# Changelog

All notable changes to the `stride-copilot-ideation` plugin are documented here.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.2.1] - 2026-06-26

### Security

- **Audit: the install path does not overwrite a project-root context file (no fix needed).** Reviewed the Copilot install path for the `AGENTS.md`-overwrite class of bug found in the OpenCode/Codex ideation installers. This plugin has no `install.sh`/`install.ps1`; it installs exclusively via `copilot plugin install https://github.com/cheezy/stride-copilot-ideation`, which places the plugin inside Copilot's managed plugin directory. `plugin.json` declares only metadata plus `agents`/`skills` pointers ‚Äî no install hook and no directive that writes any file to the user's project root ‚Äî and the plugin ships no `AGENTS.md`/context file at all. The only project-root file the README references is `.stride_auth.md`, which the user creates manually (and is told to `.gitignore`); the installer never writes it. Conclusion: safe ‚Äî no project-root context file is ever clobbered, so the managed-block guard applied to the OpenCode/Codex installers is unnecessary here.

## [0.2.0] - 2026-06-15

Ports the upstream [`cheezy/stride-ideation`](https://github.com/cheezy/stride-ideation) **v0.8.0** feature set to GitHub Copilot CLI ‚Äî six `stride-ideation-ideate` session-experience features plus a `stride-ideation-stridify` preview-and-approval gate. Every interaction is adapted to the Copilot CLI platform's question/selection primitive (never Claude Code's `AskUserQuestion`), and every new shell helper/suite ships with its PowerShell mirror for Windows parity. Additive ‚Äî `--profile=lean` (and flag-free) sessions remain compatible with v0.1.0 on the happy path.

### Added

- **Per-round section-completeness recap (`## Round recap` in the protocol skill).** Before every round ‚Äî including round 1 (all empty) and the gap-fill rounds ‚Äî the skill prints a compact recap of the seven hard-gated sections with a per-section `solid` / `thin` / `empty` status plus a one-line note of the round's target sections. Display-only narration printed to the Copilot CLI (never a question), and it never changes the gate, the round order, or the per-round question budget. Lists only the seven gated sections (never profile-exclusive optional sections), so it renders identically under every profile.
- **"I'm not sure ‚Äî propose candidates" uncertainty path (`## Uncertainty path`).** Every gated-section and profile-specific forcing question (JTBD, Why-now, MVP design) now carries a first-class choice, with phrasing identical across rounds, presented through the Copilot CLI selection primitive, that flips the skill into teaching mode: it proposes 2‚Äì4 concrete, topic-tailored candidate answers each with a one-line rationale, then the human picks, edits, or asks for a fresh batch. A proposed candidate never satisfies the gate until the human confirms it, and it is a choice *within* a question so it never breaches the per-round budget.
- **Profile recommendation when `--profile` is omitted (`stride-ideation-ideate` Step 1).** Instead of silently defaulting to lean, an omitted `--profile` runs a single recommendation (via the Copilot CLI question UI) before the rounds: a best-guess profile offered first (labeled `(recommended)` with a one-line rationale), the other three as alternatives, defaulting to lean when inference is weak. An explicit `--profile` skips the recommendation, the unknown-value fast-fail is unchanged, and a resolved `lean` is byte-for-byte identical to passing `--profile=lean`.
- **`--input <file>` brain-dump seed (`stride-ideation-ideate` Step 1 + new Step 4c).** A new optional flag (both `--input <path>` and `--input=<path>` forms), distinct from `--continue`: the skill reads the freeform notes file **read-only** via the platform file-read tool, pre-populates draft sections wherever the notes clearly map to a gated section, then focuses the rounds on the gaps. The input file is never written, moved, or `git add`-ed, and its contents are treated as untrusted prose. `--input` and `--continue` are independent and composable; seeded content never satisfies a hard gate on its own.
- **Intra-session draft autosave and resume (new `lib/draft.sh` + `lib/draft.ps1`).** The in-progress draft is autosaved after every round to a **gitignored** scratch file under `.stride/`, so an interruption mid-session is recoverable. On start, an unfinished draft for the **same slug** is detected (`sti_draft_find` / `Sti-DraftFind`) and the human is offered resume-or-fresh via the platform question UI; on a successful commit the scratch file is deleted. The pure-shell `lib/draft.sh` (mirroring `lib/filename.sh`) and its PowerShell mirror `lib/draft.ps1` provide `sti_draft_path/find/save/load/exists/clear` ‚Äî resume keys on the slug with a dash-delimited glob so `auth` never matches `oauth`, and empty/partial drafts safely fall back to a fresh session. The scratch file never holds the Stride API token or any secret.
- **Advisory reviewer findings surfaced as an explicit human decision (`## Reviewer pass`).** When the `requirements-reviewer` returns `verdict: "issues_found"`, the skill now presents the findings through the Copilot CLI selection primitive ‚Äî one selectable option per finding, tagged with its `severity` (`blocking` / `advisory`) and `section` ‚Äî including an explicit **"Address none ‚Äî write as-is"** choice. The human chooses which findings feed the single refinement round. Both contracts are preserved and stated explicitly: **at most one** refinement round and the reviewer **never blocks the write** (write-as-is is always available, so even `blocking`-severity findings cannot hold the document hostage). An `approved` verdict with zero findings shows no prompt.
- **Human preview-and-approval gate before the `stridify` POST, with a `--yes` bypass (new Step 8.5 in `stride-ideation-stridify`).** Between writing/committing the batch JSON and POSTing it, the skill now renders the decomposed goal/task tree ‚Äî each goal title, its task count, its task titles, and the cross-goal claim order from `decomposition_notes` ‚Äî and requires explicit human approval (via the Copilot CLI question UI) before the POST. A new `--yes` / `--auto-approve` flag (explicit only, never inferred) bypasses the gate for scripted callers, preserving the historical fire-and-forget behavior. On decline the skill stops cleanly (exit 0) leaving the already-committed batch JSON untouched on disk ‚Äî no POST attempted. The preview reads only the on-disk JSON and never prints the API token.

### Test coverage

- **`lib/test-stridify-preview.sh` + `lib/test-stridify-preview.ps1`** (14 + 14) ‚Äî `--yes`/`--auto-approve` parse (both forms + absence), bypass-reaches-POST with no prompt, decline-no-POST + on-disk JSON byte-for-byte unchanged (SHA-asserted), approve-reaches-POST, multi-goal task-count + cross-goal claim-order render, single-goal scoped render, and a token-hygiene guard.
- **`lib/test-ideate-input.sh` + `lib/test-ideate-input.ps1`** (11 + 11) ‚Äî `--input` parse (both forms), file-exists validation (happy + missing-file), the read-only sha256 invariant, the `--input`/`--continue` precedence, and the empty-file fallback.
- **`lib/test-draft.sh` + `lib/test-draft.ps1`** (21 + 21) ‚Äî deterministic path generation, save/load round-trip, the non-empty `exists` predicate, idempotent `clear`, `find` (same-slug-only, `auth`-vs-`oauth` non-collision, latest-wins, empty-only ignored, absent-dir safe), the mkdir/write-failure branch, and the three edge cases (no file ‚Üí fresh, empty/partial ‚Üí fresh, two slugs ‚Üí only matching).

The bash test suite is now **155 assertions across 11 sibling suites**; the Windows PowerShell mirror suite is **114 assertions across 11 suites** ‚Äî all green. `lib/run_smoke_test.sh` (and `.ps1`) remain 10/10.

### Migration

- **No migration on the happy path.** A flag-free or `--profile=lean` `stride-ideation-ideate` session behaves as in v0.1.0 except for the additive session affordances (the recap is display-only; the uncertainty path is an extra option, never forced; the reviewer decision replaces a silent skill choice but keeps the one-round cap and never blocks the write; autosave writes only to the gitignored `.stride/` scratch path).
- **`stride-ideation-stridify` gains a human gate by default ‚Äî scripted callers must pass `--yes`.** A non-interactive `stridify` invocation that previously POSTed immediately will now pause for approval unless `--yes` / `--auto-approve` is supplied. Under `--yes` the flow is byte-for-byte identical to v0.1.0.
- **`.stride/` is the autosave scratch directory** and is already gitignored ‚Äî never committed, never holds secrets.

### Notes on the port

- **Skill activation language.** Where upstream says "Run `/stride-ideation:ideate`", this plugin says "Activate the `stride-ideation-ideate` skill". The substance of every step is preserved; only the invocation grammar changes.
- **Tool name generalization.** References to Claude Code's `Read` / `Write` / `Bash` / `Skill` / `Agent` tools are generalized to "the platform's file-read tool" / "the platform's question UI" / etc., so Copilot CLI's tool inference handles the rest. Bash snippets remain literal (they run regardless of platform).
- **PowerShell mirrors.** Every bash helper and test file has a `.ps1` mirror so Windows Copilot users get parity. The PowerShell code uses W505-reviewer-approved patterns (`Set-StrictMode Latest`, `@()` wrap for `Get-Content`, `-notcontains` for negation, no Write-Output pollution into return channels). PowerShell runtime verification deferred ‚Äî pwsh is not installed on the porting host; the same pattern was used in the reviewer-approved W784/W789 PowerShell hook mirrors in stride-gemini/stride-copilot.
- **Upstream behavior preserved.** Every load-bearing safety check ‚Äî `--profile` validation, `--continue` validation, HARD INVARIANT, 7-section gate, retry classifier, retry-exhaustion prompt-on-disk, defensive `source_spec` overwrite, never-retry-POST, source-doc read-only ‚Äî is preserved verbatim. The fixtures (which are calibration references) and the decomposer/reviewer agent bodies are byte-identical to upstream so a regression run against the same requirements docs produces structurally identical batch JSON.
- **Not on a marketplace.** `stride-copilot-ideation` is **not** published on the `stride-marketplace` (which only carries the Claude version). Install via the direct GitHub URL.
