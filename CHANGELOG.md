Initial release — port of [`cheezy/stride-ideation`](https://github.com/cheezy/stride-ideation) v0.7.0 to GitHub Copilot CLI.

### Notes on the port

- **Skill activation language.** Where upstream says "Run `/stride-ideation:ideate`", this plugin says "Activate the `stride-ideation-ideate` skill". The substance of every step is preserved; only the invocation grammar changes.
- **Tool name generalization.** References to Claude Code's `Read` / `Write` / `Bash` / `Skill` / `Agent` tools are generalized to "the platform's file-read tool" / "the platform's question UI" / etc., so Copilot CLI's tool inference handles the rest. Bash snippets remain literal (they run regardless of platform).
- **PowerShell mirrors.** Every bash helper and test file has a `.ps1` mirror so Windows Copilot users get parity. The PowerShell code uses W505-reviewer-approved patterns (`Set-StrictMode Latest`, `@()` wrap for `Get-Content`, `-notcontains` for negation, no Write-Output pollution into return channels). PowerShell runtime verification deferred — pwsh is not installed on the porting host; the same pattern was used in the reviewer-approved W784/W789 PowerShell hook mirrors in stride-gemini/stride-copilot.
- **Upstream behavior preserved.** Every load-bearing safety check — `--profile` validation, `--continue` validation, HARD INVARIANT, 7-section gate, retry classifier, retry-exhaustion prompt-on-disk, defensive `source_spec` overwrite, never-retry-POST, source-doc read-only — is preserved verbatim. The fixtures (which are calibration references) and the decomposer/reviewer agent bodies are byte-identical to upstream so a regression run against the same requirements docs produces structurally identical batch JSON.
- **Not on a marketplace.** `stride-copilot-ideation` is **not** published on the `stride-marketplace` (which only carries the Claude version). Install via the direct GitHub URL.
